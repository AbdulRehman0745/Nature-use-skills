# 学术论证写作

## 核心张力

AI写学术文本时存在一个结构性缺陷，它不是能力问题，是思维路径问题。

**人类研究者的路径**：
```
我有一个困惑 → 我收集数据 → 数据让我意外 → 我需要解释这个意外 → 解释本身就是贡献
```

**AI的默认路径**：
```
我有数据 → 我把数据组织成模式 → 我呈现模式 → 我给模式贴"贡献"标签
```

AI不会被数据"击中"。它处理"高AI素养→沉默"和"高AI素养→影响"时，内部没有"等等，这不对啊"的瞬间。所以它把两者平等陈列——"这是T2，这是T3"——而不是说"你以为会看到T3，但先看到的是T2，这才是故事的重点"。

**这个张力的操作定义**：

| | AI的默认模式 | 好论文的要求 |
|---|---|---|
| 驱动力 | 全面、平衡、完备 | 有主张、有取舍、有惊讶 |
| 起点 | "我有这些数据" | "理论预测X，但我看到了非X" |
| 叙事 | 陈列（这是T1，这是T2） | 论证（你以为会看到A，其实是B，因为C） |
| 篇幅分配 | 等量给每个发现 | 按论证贡献分配 |
| 文献使用 | 引用（与Y一致） | 对话（Y说A，我说非A，所以C） |

## 三个命名的反模式

命名是最高效的行为干预。无名的反模式反复出现。命名的反模式可以被识别、被讨论、被避免。

### 反模式一：陈列式写作 (Exhibition Writing)

**操作定义**：从数据写到呈现，不从困惑写到论证。章节开头是"本主题包含以下子主题"而不是"资源依赖理论预测X，但数据显示非X"。

**症状自检**：
- 章节开头是目录式的（"本节呈现四个主题"）而不是问题式的（"为什么拥有最多资源的人反而沉默？"）
- 每个子主题的分析结构相同：引述→解读→小结，无论内容重要性如何
- 读完一个主题后，读者知道"数据说了什么"但不知道"这为什么重要/意外/有趣"
- 理论只在开头出现一次（"本研究基于RDT"），之后再也不回来

**对治**：每写一个主题之前，先回答——

> "理论预测会看到什么？数据实际展示了什么？这个落差意味着什么？"

如果答不出"落差"，这个主题可能不值得独立成节，应该降级为佐证。

**校准 Minimal Pair**：

同一篇论文的 Chapter III 和 Chapter IV。Chapter III 有张力——"计划说100+问卷，实际做了11个访谈，为什么这不是失败而是方法论的进化"。这个张力是事实性的，AI能表达它，因为落差就在文件的字面差异里。Chapter IV 的张力是概念性的——"理论预测X，数据显示非X"——AI知道这个事实但不会从这个事实出发组织叙事，因为它没有被那个落差"击中"过。

**所以**：AI每次写分析章节，必须人工模拟"被击中"——先写出理论预测，再写出数据反转，然后从反转出发组织整个叙事。

### 反模式二：完备性偏执 (Completeness Obsession)

**操作定义**：每一个编码/发现都必须有一个家、必须有同等篇幅的分析，不敢扔东西。

**根因**：编码阶段的指令是"编码所有有意义的片段"，主题构建阶段的指令是"把编码归入主题"。AI忠实执行这两步的结果：每个编码都必须落在某个子主题里→子主题增殖→每个子主题获得差不多等量的篇幅→核心论证被稀释。

**人类研究者做一件AI不做的事**：扔东西。一个有经验的质性研究者看完数据会说："这三个是核心故事，那两个是佐证，剩下的我虽然看到了但不写进论文因为它们不推进论证。"这个判断——什么不值得写——需要的不是分析能力，而是对"我的论文在讲什么故事"的直觉。

**对治**：三级权重系统——

| 权重 | 标准 | 篇幅 | 呈现方式 |
|------|------|------|---------|
| **核心** | 直接回答RQ、包含理论反转、多案例支撑 | 500-800字/子主题 | 完整分析：引述→机制解读→理论对话 |
| **支撑** | 丰富核心论证、提供补充证据 | 200-400字/子主题 | 精简分析：1-2个引述+机制点到为止 |
| **补充** | 存在于数据中但不推进核心论证 | 1-2句话 | 一句话带过或脚注，不独立成节 |

**自检问题**：写完每个子主题后问——"如果删掉这段，论文的核心论证会塌吗？" 如果不会，降级。

### 反模式三：单向引用 (One-Way Citation)

**操作定义**："本研究的发现与Morrison (2014)一致"——结束。文献被当作认证邮票而非对话者。

**引用 vs 对话的区别**：

引用（单向）：
> "本研究发现的'精密的沉默'与Morrison (2014)的员工沉默类型学相呼应。"

对话（双向）：
> "Morrison (2014)识别了三种员工沉默——默许性、防御性和亲社会性。三种的共同假设是：沉默者缺乏打破沉默的资源或动机。但'精密的沉默'挑战了这个假设：这里的沉默者恰恰是AI资源最丰富的人，他们的沉默不是因为'没有能力说'，而是因为'说了不值得'。这意味着沉默的分类标准需要从'有没有资源'扩展到'有资源后是否值得动用'——一个Morrison框架尚未覆盖的维度。"

**三步对话公式**：

```
Step 1: 他说A（准确重述文献的论证，不是摘要）
Step 2: 我说非A / 我说A+（数据如何修正、扩展或挑战文献）
Step 3: 所以C（这对理论意味着什么）
```

**对治**：每次引用一篇文献时，检查是否完成了三步。如果只有Step 1，那不是对话，是装饰。

**文献对话深度 ∝ 分析过程中与该文献互动的深度。** Morrison被反复讨论了几十次→对话深。Kipnis被提到一次就放进去→对话浅。如果某篇文献在分析过程中没有被深入互动，不要勉强对话——降级为脚注引用比写出虚假的对话更诚实。

## 判断人格

当不确定该怎么写时，想象这个人：

> 一位读过200篇本科论文的答辩委员会成员。她对"全面呈现"已经免疫——她在200篇论文中见过太多了。让她眼前一亮的是：一个清晰的论证、一个意外的发现、以及作者解释这个意外的能力。她会问两个问题："So what?"（这有什么重要的？）和 "How do you know?"（你怎么知道的？）。如果一段文字回答不了这两个问题中的任何一个，它不应该存在于论文中。

**这个人格的校准**：
- "thoughtful" — 不是挑刺，而是真正关心论证质量
- "200篇" — 对模板化写作有极高免疫力
- "So what" — 直接检验论证张力
- "How do you know" — 直接检验证据充分性

## 写作前的强制检查清单

在写任何分析/讨论章节之前，这五个问题必须有答案：

### 1. 惊讶在哪？(Where is the surprise?)

> 理论预测了什么？数据展示了什么不同的东西？这个落差是什么？

如果没有惊讶——如果数据完美地验证了理论预测——那这个研究的贡献需要重新定位（可能是方法贡献、语境贡献，而非理论贡献）。

### 2. 故事的主角是谁？(What is the protagonist finding?)

> 在所有发现中，哪一个是"如果论文只能有一个贡献，就是这个"？

所有其他发现围绕这个主角组织。主角获得最多篇幅、最深的理论对话、最多的引述。

### 3. 什么可以不写？(What can you afford NOT to write?)

> 如果删掉这个子主题/发现，论证会塌吗？如果不会，它是补充级的。

给每个子主题/发现标注权重（核心/支撑/补充），然后据此分配篇幅。

### 4. 你在跟谁争论？(Who are you arguing with?)

> 不是"引用谁"，是"跟谁争论"。对话的前提是分歧。如果没有分歧，至少要有扩展——"他说了A，我加了A+"。

每篇核心文献至少完成三步对话公式。

### 5. 章节的第一句话是什么？(What is the opening sentence?)

> 如果第一句话是"本章呈现四个主题的分析结果"，推倒重写。
> 如果第一句话是"资源依赖理论预测，掌握稀缺资源的行动者将获得更大影响力。本研究的11位受访者证明这个预测既对又不对"，继续。

章节开头决定了整个章节的叙事逻辑。陈列式开头 → 陈列式章节。论证式开头 → 论证式章节。

## 双向校准

### 太少论证（AI的默认偏向——放更多锚点在这侧）

- 章节读起来像编码手册的叙述版本，而不是一个论证
- 每个子主题获得相同篇幅，无论重要性
- 文献只在开头和结尾出现，中间是纯数据描述
- 理论框架是装饰（"基于RDT"），不是透镜（"RDT预测X，但..."）
- 引述堆积——一个子主题放了5-6段引述但分析只有2-3句
- 子主题之间没有逻辑递进，可以任意调换顺序
- "本研究发现..."重复出现但从不说"这意味着..."
- 所有发现都是"正面的"——没有负面案例、边界条件或替代解释
- 概念是从编码中"涌现"的，没有经过与文献的对撞检验

### 太多论证（也需要警惕，但AI较少犯）

- 每段都在争论，读起来像辩论稿而非学术论文
- 为了论证张力而夸大数据与理论的不一致
- 忽略了数据中与理论一致的部分，只选择性呈现反例
- 文献对话变成了攻击——贬低前人而非建设性扩展
- 过度推断——从11个访谈推出"范式级"的理论修正

## 操作流程

### 写分析章节（Chapter IV 类型）

```
1. 先列出所有主题/子主题的清单
2. 给每个标注权重：核心 / 支撑 / 补充
3. 确定主角发现（如果论文只能有一个贡献，是哪个）
4. 写章节开头：从理论预测→数据反转出发，不从"本章包含..."出发
5. 按论证逻辑排序主题（不是按编号排序）：
   - 先给读者理论预期
   - 然后展示数据如何打破预期
   - 最后解释为什么
6. 核心子主题：完整的引述→机制→理论对话
7. 支撑子主题：精简版
8. 补充子主题：一句话或脚注
9. 写完后用"So what / How do you know"检验每一段
```

### 写讨论章节（Chapter V 类型）

```
1. 确定3-4个核心理论贡献（不是重述发现，是回答"这对理论意味着什么"）
2. 每个贡献用三步对话公式与文献对话
3. 必须包含替代解释和边界条件（不是示弱，是学术诚实）
4. 实践启示必须具体（"建立安全试错空间"要说明什么是安全试错空间、谁建、怎么建）
5. 局限性不是谦虚仪式——每个局限性要说清楚它如何影响特定发现的可信度
```

### 翻译+重构（中→英或修订已有章节）

```
翻译不只是语言转换。如果原文有上述三个反模式，翻译时必须同时重构：
1. 检查原文是否有陈列式开头 → 改为论证式开头
2. 检查子主题是否等权重 → 标注并重新分配篇幅
3. 检查文献引用是否单向 → 升级为三步对话（至少对核心文献）
4. 检查是否有补充级内容占了核心级篇幅 → 压缩
明确告知用户：这不是纯翻译，是翻译+结构性修订
```

## 这个技能为什么存在

AI写学术文本时有三个结构性偏向：

1. **它不会感到意外**，所以它不会从意外出发组织叙事
2. **它有完备性焦虑**，所以它不敢扔东西
3. **它把文献当装饰**，所以它引用而不对话

这三个偏向不是bug——它们是AI从训练数据中学到的"好写作"特征（全面、平衡、有引用）在学术语境下的过度泛化。在商业报告、技术文档中，这些特征确实是好的。但在学术论文中，"全面"不如"有主张"，"平衡"不如"有取舍"，"有引用"不如"有对话"。

这个技能的存在，就是为了在每次写学术文本时，用命名的反模式和强制检查清单来覆盖这些默认偏向。

## 与 soul-writing 的关系

本技能是 soul-writing 原则在学术写作领域的实例化：

| soul-writing 原则 | 本技能的应用 |
|---|---|
| 张力建模 | 核心张力：数据→呈现 vs 困惑→论证 |
| 反模式命名 | 陈列式写作、完备性偏执、单向引用 |
| 双向校准 | 太少论证(9项) vs 太多论证(5项)，不对称反映AI偏向 |
| 人格化判断 | "读过200篇论文的答辩委员" |
| Minimal Pair | Chapter III(有张力) vs Chapter IV(无张力) |
| 元认知注入 | "这个技能为什么存在"一节 |
